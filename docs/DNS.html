Domain Name System (DNS)


Most people in the world don’t really worry too much about learning the IP addresses of the devices they are trying to connect to. This is thanks to the Domain Name System (DNS); DNS is in charge of associating a domain name like www.google.com to its IP address. As you can imagine with billions of devices connected to a network a single DNS server will not be able to handle the load; therefore, DNS was designed to follow a distributed hierarchy and are divided into zones. Each zone is in charge of the major domains (.org, .net, .gov, .com, etc.)


How does it work?


When you try to reach Google, your computer sends a request to a DNS server, if the server recognizes that domain name it will reply with the corresponding IP address. On the other hand, if your DNS server doesn’t recognize the domain name then it will forward the request to other DNS servers until one of them gets the answer, then your server will save it on its cache for any future references and reply with the IP address.


DNS Spoofing


Because DNS was created open to the public, its susceptible attacks. One of the most common attacks is DNS spoofing. In this scenario a hacker gets access to a DNS server and changes an entry with a different IP address. This results in any request that hit that DNS server to be routed to the wrong IP address which could contain malicious content.
