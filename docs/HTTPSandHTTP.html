HTTP & HTTPS


When you open a web browser and type in a URL (Uniform Resource Locator) your computer sends a request to a server that could be located anywhere in the globe. Within milliseconds this server replies with a website. During this process both the client and server computer are using the HyperText Transfer Protocol (HTTP) to communicate with each other. HTTP is the language used between web browsers and servers to manage the sending and receiving of web files over the internet


GET Request


This communication process is very straightforward, the client sends a GET request which is formed by the word GET and the name of the document the client needs from the server and the server replies with such document.


POST Request


However, when you are visiting a website, you are not always just requesting information. Sometimes you are also sending it via a form or a survey. Whenever you send information to a server you are actually sending an HTTP POST request. For example, when you enter your login information in a website, you send a POST request to the server with your login data.

HTTPS

As you already know, the internet is open to the public, it shares connections and the information sent within these connections is in plain text.This enables hackers to be able see the information you send over the internet. In order to prevent this, websites request your browser to communicate in a secure channel using the Secure Socket Layer (SSL) and its successor Transport Layer Security (TLS). HTTPS, also known as HyperText Transfer Protocol Secure, is basically the HTTP protocol using the SSL & TLS protocols to encrypt your data communications when navigating through the web.


Certificates

When a website asks your browser to establish a secure connection, it provides a certificate that proves its authenticity. A certificate is generated by a Certificate Authority (CA). CAs are trusted entities that verify the identity of a website. Similarly to the government verifying your identity by providing a license.
